# Phase 3 검증 가이드 (Commit 11~14)

타임라인 슬라이더 + 시간 범위별 엔티티 필터 + Cesium Clock 재생까지 적용된 상태입니다.  
아래 순서로 확인한 뒤, 문제 없으면 **수동으로 commit** 하세요.

## 1. 의존성 설치

```bash
cd frontend
pnpm install
```

- `vis-timeline`, `vis-data`가 `package.json`에 추가되어 있으므로 설치됩니다.
- 설치 후 `pnpm start`로 개발 서버를 띄웁니다.

## 2. 개발 서버로 확인 (권장)

```bash
cd frontend
pnpm start
```

- **화면 하단**에 **타임라인 슬라이더**(vis-timeline)가 보여야 합니다.
- 타임라인에 **이벤트 5개**가 점/바 형태로 표시되고, **2026-02-28** 구간이 보여야 합니다.
- **타임라인을 드래그하거나 줌**하면 보이는 시간 구간이 바뀌고, **지도 위 마커가 그 구간에 해당하는 이벤트만** 보여야 합니다.
- **왼쪽 하단**에 **▶ 재생 / ⏸ 일시정지 / ⏹ 처음으로** 버튼과 **UTC 시각**이 보여야 합니다.
- **▶ 재생**을 누르면 시간이 진행되면서, **그 시각까지 발생한 이벤트만** 지도에 순서대로 나타나야 합니다.
- **⏹ 처음으로**를 누르면 00:00으로 돌아가고, 타임라인 전체 구간의 이벤트가 다시 보여야 합니다.

## 3. 빌드 확인

```bash
cd frontend
pnpm run build
```

- 에러 없이 `build/`가 생성되는지 확인합니다.

## 4. 체크리스트 (Phase 3) — 항목별 확인 방법

### ☐ ① 하단에 vis-timeline 타임라인 UI가 표시됨

- `pnpm start` 후 브라우저에서 화면 **맨 아래**를 보세요.
- 회색/어두운 배경의 **가로 막대 타임라인**과 그 위에 **색깔 있는 이벤트 바**가 보이면 통과입니다.

---

### ☐ ② 타임라인 범위가 2026-02-28 (이란 공습 기간)으로 설정됨

- 타임라인 **축(시간 눈금)** 을 보세요.
- **날짜가 "2026-02-28"** 로만 나오고, 그날 00:00 ~ 24:00(또는 23:59) 구간만 보이면 됩니다.
- 타임라인을 **왼쪽/오른쪽 끝까지 드래그**해도 2026-02-28 밖으로 나가지 않으면 범위가 올바르게 잡힌 것입니다.

---

### ☐ ③ 타임라인 구간 변경 시 해당 구간 이벤트만 지도에 표시됨

1. **처음 상태**: 타임라인을 건드리지 않고, 지도에는 **마커 5개**가 모두 보여야 합니다.
2. **타임라인을 좁게**: 타임라인에서 **마우스로 드래그**해 보이는 구간을 **14:00 ~ 15:00** 정도로만 좁히세요.
3. **지도 확인**: 지도 위 마커가 **그 구간(14:00~15:00)에 해당하는 이벤트만** 남고, 나머지는 사라져야 합니다.  
   - 샘플 데이터 기준: 14:30, 14:45, 15:00 이벤트만 보이고, 15:30·16:00 이벤트는 보이지 않아야 합니다.
4. **다시 넓게**: 타임라인을 **줌 아웃**하거나 드래그해 **전체 24시간**이 다시 보이게 하세요.
5. **지도 확인**: 마커 **5개가 다시 모두** 보이면 통과입니다.

---

### ☐ ④ 재생 버튼으로 Cesium Clock 연동, 시간에 따라 이벤트가 순차 표시됨

1. **⏹ 처음으로** 버튼을 한 번 눌러 **00:00:00** 상태로 맞춥니다.
2. **지도 확인**: 이때는 **아무 마커도 없거나** 00:00 이전 이벤트만 있어야 합니다. (샘플 데이터는 14:30부터라 보통 **마커 0개**)
3. **▶ 재생** 버튼을 누릅니다.
4. **왼쪽 하단 UTC 시각**이 **00:00:00 → 00:01:00 → …** 처럼 **실제로 증가**하는지 확인합니다.
5. **14:30 전후**가 되면 **첫 번째 마커**가 지도에 나타나야 하고, 시간이 더 지나면 **두 번째, 세 번째…** 순서대로 나타납니다.
6. **⏸ 일시정지**를 누르면 시각이 멈추고, 그 시점까지 나타난 마커만 유지됩니다.
7. **⏹ 처음으로**를 다시 누르면 시각이 00:00으로 돌아가고, 지도에는 다시 **전체 구간** 이벤트(마커 5개)가 보여야 합니다.
